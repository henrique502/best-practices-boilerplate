## Features

- 🔒 **Security-first** — OWASP-aligned guidelines baked in from day one
- 🏗️ **TypeScript strict mode** — no `any`, no shortcuts
- ⚛️ **React patterns** — component architecture that scales
- 🔀 **Git workflow** — conventional commits and clean branching
- 📐 **EditorConfig** — consistent formatting across any editor

## Getting Started

1. Clone this repository
2. Open it in your editor (VS Code / Kiro recommended)
3. The `.kiro/steering/` files will automatically guide AI-assisted development

```bash
git clone https://github.com/your-user/best-practices-boilerplate.git
cd best-practices-boilerplate
```

## What's Included

| Steering File | Purpose |
|---|---|
| `typescript-best-practices.md` | Strict typing, naming conventions, error handling |
| `react-best-practices.md` | Component patterns, hooks, state management |
| `security-best-practices.md` | Input validation, secrets management, OWASP Top 10 |
| `git-best-practices.md` | Conventional commits, branching strategy |
| `editorconfig.md` | Formatting rules derived from `.editorconfig` |
| `use-bash.md` | Shell environment standardization (Git Bash) |

## Contributing

1. Create a feature branch: `git checkout -b feature/my-improvement`
2. Follow the conventions defined in `.kiro/steering/`
3. Commit using conventional format: `feat(scope): description`
4. Open a pull request
